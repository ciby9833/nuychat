四、你现在设计里最大风险（重点）

⸻

⚠️ 风险 1：Agent 和 Skill 混淆

很多系统死在这里：

Agent = Skill（错）


⸻

正确：

Agent = 调度者
Skill = 执行者


⸻

⚠️ 风险 2：Tool / Skill / Workflow 没分层

必须分清：

Tool（基础能力）
Skill（业务能力）
Workflow（流程）


⸻

⚠️ 风险 3：AI 座席 vs Orchestrator 没分清

👉 很多人做成：

一个 AI 什么都干


⸻

正确：

AI Agent（前台）
↓
Orchestrator（后台调度）

标准架构（Claude / OpenAI / Intercom）

⸻

Layer 1：Agent Layer

Tool Runner

LLMs




目标

构建 AI Native 客服系统

实现：
- AI座席（具备明确能力边界）
- 人工座席 + AI Copilot
- 通用智能引擎（Knowledge / Skills / Tools / Orchestration）


⸻

1 能力系统化（不是 prompt 驱动）
2 Agent 与 Skill 解耦
3 Tool / Skill / Workflow 分层
4 AI 负责决策，不直接操作系统
5 Orchestrator 负责复杂任务调度
