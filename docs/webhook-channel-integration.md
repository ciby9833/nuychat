# Webhook Channel Integration

`webhook` channel is the generic HTTP integration channel for third-party systems.

## Model

- Inbound webhook URL:
  - Generated by the platform
  - Derived from `API_PUBLIC_BASE + /webhook/:channelId`
  - Example: `https://api.nuychat.com/webhook/webhook-tenant-a`
- Outbound webhook URL:
  - Configured by the tenant
  - Used by NuyChat to push replies back to the third-party system
- Verify token:
  - Optional
  - Used only when the third-party platform requires a `GET` challenge flow
- Webhook secret:
  - Optional
  - Used for HMAC-SHA256 signature verification on inbound and outbound requests

## Tenant Setup

Tenant admin should configure:

1. `channel_id`
2. `outboundWebhookUrl`
3. `verifyToken` (optional)
4. `webhookSecret` (optional)

Tenant admin can view:

1. Generated inbound webhook URL
2. Current outbound webhook URL

## Inbound Flow

Third-party system sends customer messages to:

```txt
POST {API_PUBLIC_BASE}/webhook/{channelId}
Content-Type: application/json
```

If `webhookSecret` is configured, send:

```txt
X-NuyChat-Signature: sha256=<hex>
```

The signature is:

```txt
hex(HMAC_SHA256(raw_request_body, webhookSecret))
```

### Supported inbound payload

```json
{
  "id": "ext-msg-001",
  "customerRef": "customer-123",
  "displayName": "Tom",
  "text": "hello",
  "timestamp": "2026-03-12T10:00:00Z"
}
```

### Field meanings

- `id`: optional external message ID
- `customerRef`: required customer identifier in the third-party system
- `displayName`: optional display name
- `text`: message body
- `timestamp`: optional ISO timestamp

## Optional Verify Challenge

If the third-party platform requires challenge verification:

```txt
GET {API_PUBLIC_BASE}/webhook/{channelId}?hub.mode=subscribe&hub.verify_token={verifyToken}&hub.challenge=123
```

If `verifyToken` matches the tenant channel config, NuyChat returns:

```txt
123
```

## Outbound Flow

When NuyChat sends a reply through this channel, it POSTs to the configured `outboundWebhookUrl`.

### Outbound payload

```json
{
  "event": "message.outbound",
  "to": "customer-123",
  "text": "hello from NuyChat",
  "timestamp": "2026-03-12T10:01:00.000Z"
}
```

If `webhookSecret` is configured, NuyChat sends:

```txt
X-NuyChat-Signature: sha256=<hex>
```

### Expected response

The third-party system should return HTTP `200`.

Optional response body:

```json
{
  "messageId": "third-party-outbound-001"
}
```

If no `messageId` is returned, NuyChat generates one internally.

## Local Testing

For local testing:

1. Start API locally, for example `http://localhost:3000`
2. Expose it with a public HTTPS tunnel such as `ngrok` or `cloudflared`
3. Set:

```env
API_PUBLIC_BASE=https://your-temporary-public-domain.example
```

4. Give the generated inbound URL to the third-party system

## Notes

- Inbound URL is system-generated and should not be edited by tenants.
- Outbound URL is tenant-owned because it points to the tenant's external system.
- `webhook` channel is best used for CRM, legacy systems, custom apps, or partner platforms that can only integrate over HTTP.
