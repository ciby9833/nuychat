 
**目标**：规范 AI 与人工的调度分配、转人工、排队及离线处理流程，提升 AI deflection 率（AI 解决率）同时保障用户体验

## 1. 核心原则

1. **先 AI，后人工**：能用 AI 解决的优先 AI 处理
2. **透明告知**：任何调度决策都必须让用户清楚知道“为什么转/为什么不转”
3. **上下文传递**：转人工时必须带完整对话记录 + AI 总结
4. **永不无限等待**：无人工时必须提供异步方案
5. **可配置、可观测**：所有规则在后台可配置，调度结果记录到 Trace

## 2. 调度分配决策逻辑（Routing Logic）

系统在 **Semantic Router** 第一层完成初步判定，结合以下多维度规则最终决定：

| 决策维度       | 分配给 AI（优先）                          | 直接/优先分配给人工                          | 触发方式 |
|----------------|--------------------------------------------|---------------------------------------------|----------|
| **用户意图**   | FAQ、产品知识、订单查询、物流跟踪、流程说明 | 投诉、退款、取消、个性化方案、法律/隐私问题 | 语义分类 |
| **情绪状态**   | 中性/正面                                  | 负面（愤怒、抱怨、重复提问≥3次）            | Sentiment Analysis |
| **问题复杂度** | 可通过知识库/工具 1-2 轮解决               | AI 尝试 2-3 轮仍未解决，Confidence < 70%    | 多轮失败机制 |
| **业务价值**   | 标准流程                                   | VIP/高客单价用户、紧急事件、涉及金钱操作    | 客户标签 |
| **用户明确要求**| -                                          | 用户说“人工”“转人工”“我要真人”            | 关键词/语义 |
| **人工可用性** | 任意时间                                   | 人工在线且有空闲席位                        | 实时检查 |

**决策流程**：
1. Semantic Router → 判断轨道（Knowledge / Action / Clarification / Human）
2. Pre-reply-policy + 规则引擎 → 最终判定
3. 若判定为 Human Track → 执行转人工

## 3. AI 转人工业务场景及具体例子

### 场景清单（带真实对话示例）

**场景1：用户主动要求转人工**
- **业务例子**：用户咨询“我的订单怎么还没到”，聊了两轮后说“我不想跟AI说了，转人工”
- **AI 回复模板**：
  ```
  好的，我完全理解您希望与人工客服沟通。我已把您订单号【123456】、物流查询记录和我们之前的对话完整保存。
  现在为您转接人工客服，请稍等～
  ```
- **转人工后人工第一句**： “您好，我是人工客服小李，已看到您和AI的全部聊天记录，关于订单【123456】物流延误的问题，我来帮您处理。”

**场景2：AI 能力边界（多次未解决）**
- **业务例子**：用户问“这个定制产品能不能改规格”，AI 尝试用知识库和工具仍无法确认最终方案
- **AI 回复模板**：
  ```
  这个定制规格的调整需要人工审核确认。为了给您最准确的答复，我现在转接给我们的产品专员，他们会根据您的需求给出方案。
  请稍等，我马上为您转接。
  ```

**场景3：负面情绪 / 投诉**
- **业务例子**：用户连续说“你们物流太慢了”“上次就延误，这次又这样”
- **AI 回复模板**：
  ```
  我感受到您对物流延误很着急，为了更快解决您的问题，我马上转接给人工客服，他们会优先处理您的投诉并给出补偿方案。
  对话记录已同步，请稍等。
  ```

**场景4：高价值 / 敏感业务**
- **业务例子**：用户要求“帮我全额退款”或“VIP 会员权益调整”
- **AI 回复模板**：
  ```
  涉及金额操作的退款请求需要人工审核。我已整理好您的订单信息和退款理由，现在转接给财务专员。
  预计 1 分钟内接通。
  ```

**场景5：复杂多轮未闭环**
- **业务例子**：用户连续问了 5 轮关于“合同条款修改”，仍未达成一致
- **AI 回复模板**：
  ```
  我们已经聊了好几轮，为了不耽误您时间，我现在把全部对话记录转给人工客服，他们会继续跟进并给出最终方案。
  ```

## 4. 排队处理机制

| 排队情况         | 处理方式                                      | 用户可见提示示例 |
|------------------|-----------------------------------------------|------------------|
| 轻度排队（≤3人） | 立即显示预计等待时间                          | “当前排队 2 人，预计等待 1 分 30 秒” |
| 中度排队         | AI 继续服务 + 加入优先队列                    | “我已把您加入优先队列，人工客服空闲后会第一时间联系您” |
| 重度排队 / 高峰  | 提供异步选项（工单、预约回调）                | “当前排队人数较多，我建议先帮您创建工单，人工会在 30 分钟内处理” |
| VIP / 紧急       | 插队机制                                      | “您是 VIP 用户，我已为您插队，预计 30 秒内接通” |

## 5. 无人工在线（离线）处理场景

**触发条件**：工作时间外、人工全部离线、或坐席全部忙碌

**核心流程**：
1. 立即告知当前状态
2. 提供 3 个异步选项
3. AI 继续最大化帮助
4. 自动创建工单并承诺时效

**具体例子及回复模板**：

**例子1：夜间普通咨询**
- **业务例子**：晚上 22:00 用户问“明天能发货吗？”
- **AI 回复**：
  ```
  我已为您查询，目前人工客服团队处于非工作时间（工作日 09:00-18:00）。
  我可以：
  1. 立即帮您创建工单，明天上午 09:30 前人工处理
  2. 用AI继续为您解答（大部分问题我都能处理）
  3. 预约明天人工客服电话回访

  请回复数字 1/2/3，或直接告诉我您的需求，我先帮您看看。
  ```

**例子2：紧急投诉（离线）**
- **业务例子**：用户半夜投诉“订单丢失”
- **AI 回复**：
  ```
  我理解这个问题很紧急。目前人工已下班，我立即为您创建【紧急工单】，并标注优先级。
  明天上午 09:00 人工客服上班后会在 30 分钟内联系您。
  您方便留下手机号吗？我安排优先回电。
  ```

**例子3：用户坚持要人工（离线）**
- **AI 回复**：
  ```
  好的，我已记录您的强烈需求。目前人工离线，我会立即创建加急工单并同步全部对话。
  人工明天 09:00 上班后会第一时间处理并联系您。
  工单编号：T20260413-887766，已发送至您的账号。
  ```

## 6. 系统实现建议（对接 NuyChat 重构）

- **Semantic Router** 新增 `HumanTrack` 轨道
- **orchestrator.service.ts** 增加 `checkHumanAvailability()` 调用
- **answer-composer.service.ts** 增加 Offline / Handoff 专用回复模板引擎
- 转人工时自动调用工单创建接口，并携带：
  - 完整 chatHistory
  - AI summary
  - confidence score
  - trigger reason
- 后台配置：工作时间表、技能组、溢出规则、模板库

## 7. 最佳实践 & 验收标准

- **Deflection 率目标**：AI 解决率 ≥ 75%
- **转人工成功率**：上下文传递 100%
- **用户满意度**：转人工后 CSAT ≥ 4.5 分
- **监控指标**：转人工原因分布、离线工单创建量、排队平均等待时间

**验收用例**（必须测试）：
1. 用户主动说“转人工” → 成功转接
2. 负面情绪多轮 → 自动转人工
3. 夜间 23:00 咨询 → 提供工单 + AI 继续服务
4. VIP 用户离线投诉 → 创建紧急工单 + 预约回调
